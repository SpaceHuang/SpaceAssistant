# LLM Wiki 使用手册

## LLM Wiki 是什么？

LLM Wiki 是 SpaceAssistant 内置的个人知识库。它能帮你把收集的资料（文章、笔记、文档）整理成结构化的知识网络，让 AI 能够「记住」你的知识，并在需要时快速调用。

### 和传统搜索有什么不同？

| 传统搜索 / RAG | LLM Wiki |
|----------------|----------|
| 每次从原始文档临时检索片段 | 知识**持续积累**，随使用越来越好用 |
| 分散的片段，难以形成整体认知 | 页面之间**互相关联**，形成知识网络 |
| 不知道你积累了什么 | 像百科全书一样可以**浏览和探索** |

## 快速入门

### 第一步：启用并初始化

1. 打开设置（右上角设置图标）
2. 找到「Wiki」选项
3. 开启「启用 LLM Wiki」
4. 点击「初始化 Wiki」

初始化后，你的`工作目录`下会创建一个 `llm-wiki/` 文件夹，这是你的知识库根目录。

### 第二步：放入资料

将你收集的文章、笔记放到 `llm-wiki/raw/` 目录下。支持 `.md` 和 `.txt` 格式的纯文本文件。

### 第三步：开始使用

在聊天框中输入命令即可使用：

```
/wiki ingest raw/我的笔记.md    ← 收录资料
/wiki query xxx是什么？         ← 基于知识库回答
/wiki lint                      ← 检查知识库健康状况
```

## 三大核心操作

### 收录（Ingest）

**作用**：把原始资料「消化」成结构化的知识页面。

当你执行收录时，AI 会：
- 阅读你放入的原始资料
- 自动创建或更新相关知识页面（实体、概念、主题）
- 更新知识库的目录和日志

**命令格式**：

| 命令 | 说明 |
|------|------|
| `/wiki ingest raw/文件.md` | 收录指定文件 |
| `/wiki 摄取 raw/文件.md` | 同上（中文别名） |
| `/wiki 提取 raw/文件.md` | 同上（中文别名） |
| `/wiki ingest --all` | 批量收录 raw 目录下所有文件 |

**从文件列表收录**：
- 在文件列表中右键任意文件
- 选择「收录到 Wiki」
- 应用会自动复制到 `raw/` 并开始收录

### 查询（Query）

**作用**：基于已整理的知识库回答问题。

当你提问时，AI 会：
- 先查看知识库的目录定位相关内容
- 综合多个相关页面的信息回答
- 在回答中引用具体知识页面的路径

**命令格式**：

| 命令 | 说明 |
|------|------|
| `/wiki query 我的项目用到了哪些技术？` | 提问 |
| `/wiki 我想了解关于 xxx 的内容` | 省略 query 直接提问 |

**保存优质回答**：
如果你觉得 AI 的回答很好，可以点击回答旁边的「归档到 Wiki」按钮，将其保存为新的知识页面，供以后参考。

### 检查（Lint）

**作用**：检查知识库的健康状况，发现潜在问题。

AI 会检查：
- **矛盾**：不同页面的说法互相冲突
- **过时**：新资料已替代旧结论但未更新
- **孤儿页**：没有任何页面引用的孤立页面
- **缺失链接**：提到但还没创建对应页面的内容

**命令格式**：

| 命令 | 说明 |
|------|------|
| `/wiki lint` | 健康检查（只报告问题） |
| `/wiki lint --fix` | 检查并自动修复问题 |

## 目录结构

LLM Wiki 的文件夹结构如下：

```
llm-wiki/                          ← 知识库根目录
├── SCHEMA.md                      ← 维护规范（告诉 AI 如何工作）
├── raw/                           ← 原始资料（只读）
│   ├── 2026-01-01-文章1.md
│   └── 2026-01-02-文章2.md
└── wiki/                          ← AI 维护的知识库
    ├── index.md                   ← 知识库目录（从这里开始浏览）
    ├── log.md                     ← 操作日志
    ├── entities/                  ← 实体页（人物、组织、产品等）
    ├── concepts/                  ← 概念页（术语、定义等）
    ├── topics/                    ← 主题页（综合分析、综述）
    ├── sources/                   ← 原始资料摘要
    └── queries/                   ← 保存的问答记录
```

### 各目录用途

| 目录 | 内容 | 谁写 |
|------|------|------|
| `raw/` | 原始文章、笔记 | 你（用户） |
| `wiki/` | AI 整理的知识页面 | AI |
| `SCHEMA.md` | 维护规则和约定 | AI 和你共同维护 |

### 浏览知识库

在「文件」面板下方，你可以看到「LLM Wiki」分区：

- 点击 `index.md` 查看知识库的整体结构
- 通过链接在页面之间跳转
- 点击任意页面可在右侧预览

## 常见问题

### 问：LLM Wiki 和 Skills 有什么区别？

Skills 是告诉 AI「怎么做事」的规范（比如代码审查流程）。LLM Wiki 是 AI 维护的「知识库」，让 AI 能够调用已积累的知识。

两者可以同时使用，互不干扰。

### 问：放在 raw 目录的资料会被修改吗？

不会。`raw/` 目录是**只读**的，AI 只能阅读，不能修改。如果需要修正格式，AI 会提示你确认。

### 问：支持 PDF、Word、图片吗？

**首版仅支持纯文本文件**（`.md`、`.txt`）。PDF、Word、图片等需要先转换为 Markdown 格式再放入。

### 问：知识库会越来越大，上下文不够用怎么办？

LLM Wiki 的设计就是解决这个问题的。Query 时 AI 会先读 `index.md` 定位需要的内容，而不是每次都读全部资料。对于超大型知识库，还可以通过 `/wiki lint` 发现并整理冗余内容。

### 问：多个会话可以同时使用 Wiki 吗？

可以，但需要注意：**并行收录可能产生编辑冲突**。建议一个 Wiki 只在一个会话中使用，或通过 Git 等工具管理版本。

### 问：Wiki 保存在哪里？可以 Git 管理吗？

Wiki 保存在你的工作目录下的 `llm-wiki/` 文件夹，是普通的 Markdown 文件，完全可以用 Git 管理版本。

## 命令速查表

| 功能 | 命令 |
|------|------|
| 初始化 Wiki | 设置中开启「启用 LLM Wiki」|
| 收录资料 | `/wiki ingest <path>` |
| 收录资料（中文） | `/wiki 摄取 <path>` 或 `/wiki 提取 <path>` |
| 批量收录 | `/wiki ingest --all` |
| 提问 | `/wiki query <问题>` 或 `/wiki <问题>` |
| 健康检查 | `/wiki lint` |
| 检查并修复 | `/wiki lint --fix` |
| 查看状态 | `/wiki status` |
| 查看帮助 | `/wiki help` |

## 提示与技巧

1. **定期收录**：遇到好文章就随手收录，知识库会越来越丰富
2. **定期 Lint**：运行 `/wiki lint` 保持知识库健康
3. **善用归档**：看到好的回答就「归档到 Wiki」，积累多了很有用
4. **从 index 开始**：浏览知识库从 `index.md` 开始，通过链接探索
5. **命名规范**：raw 文件建议用「日期 + 标题」命名，如 `2026-06-01-量子计算入门.md`
