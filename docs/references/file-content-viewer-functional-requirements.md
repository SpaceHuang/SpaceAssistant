## 1. 背景与目标

WorkAgent 的文件内容查看器（集成于 DetailPanel）是用户查看文件内容的核心组件。当前已实现基础的文本展示能力，并逐步扩展了语法高亮、Markdown 渲染切换、查找替换、文件导出等功能。

**目标：** 整合现有功能定义，形成完整的功能需求文档，为后续开发和维护提供统一的参考依据。

---

## 2. 功能需求

### 2.1 文件打开入口

文件内容查看器通过详情面板展示文件内容，以下入口可触发文件打开：

#### 2.1.1 文件树（FileTree）

| 属性 | 说明 |
|------|------|
| 位置 | 左侧工作区面板的「文件」标签页 |
| 入口组件 | `FilePanel.tsx` → `FileTree.tsx` |
| 触发方式 | 点击文件树中的任意文件节点 |
| 回调链 | `FileTree.onSelectFile` → `AppContext.openFile` → `selectedFile` 状态更新 |

**交互流程：**
```
用户点击 FileTree 中的文件
  → onSelectFile(filePath) 回调触发
  → App.openFile(filePath) 执行
  → setSelectedFile(filePath) 状态更新
  → DetailPanel 渲染文件内容
```

#### 2.1.2 消息文件卡片（MessageItem）

| 属性 | 说明 |
|------|------|
| 位置 | 聊天消息区域的消息卡片 |
| 入口组件 | `MessageItem.tsx`（`role === 'file_written'` 时渲染） |
| 触发方式 | 点击消息中显示的文件卡片按钮 |
| 回调链 | `MessageItem.openFile(filePath)` → `AppContext.openFile` → `selectedFile` 状态更新 |

**交互流程：**
```
用户点击消息中的文件卡片按钮
  → openFile(message.filePath) 执行
  → setSelectedFile(filePath) 状态更新
  → DetailPanel 渲染文件内容
```

#### 2.1.3 状态管理

- **状态定义**：`selectedFile: string | null`，存储当前打开文件的绝对路径
- **状态共享**：通过 `AppContext` 在组件间共享 `openFile` 方法
- **关闭文件**：调用 `onClose={() => setSelectedFile(null)}` 关闭文件并恢复占位符状态

### 2.2 工具栏功能

#### 2.2.1 工具栏组成

文件内容查看器的工具栏（`FileToolbar`）包含以下功能按钮：

| 按钮 | 图标 | Tooltip | 功能说明 |
|------|------|---------|----------|
| 视图切换 | 文字按钮 | 渲染/代码 | Markdown 文件的代码模式与渲染模式切换 |
| 用默认编辑器打开 | `external_link_line` | 用默认编辑器打开 | 调用系统默认应用打开当前文件 |
| 查看所在目录 | `folder_open_line` | 查看所在目录 | 在文件资源管理器中打开并高亮文件 |
| 刷新 | `refresh_1_line` | 刷新 | 重新读取文件内容 |
| 导出为... | `export_line` / `download_line` | 导出为... | 仅 Markdown 文件显示，支持导出 PDF/docx |
| 关闭 | `close_line` | 关闭 | 关闭文件查看，返回占位符状态 |

#### 2.2.2 工具栏样式

- **高度**：36px
- **布局**：flex 布局，视图切换按钮居左，其余按钮居右
- **按钮尺寸**：28×28px，圆角 6px
- **背景**：与面板背景一致，底部边框分隔

### 2.3 内容查看功能

#### 2.3.1 文件类型支持

根据文件后缀进行分类处理：

| 类型 | 支持后缀 | 处理方式 |
|------|----------|----------|
| 纯文本 | `.txt`, `.log`, `.csv`, `.tsv` | 直接读取渲染 |
| Markdown | `.md`, `.mdx`, `.rst` | 支持代码/渲染双模式 |
| Web 代码 | `.html`, `.css`, `.scss`, `.js`, `.jsx`, `.ts`, `.tsx`, `.vue`, `.svelte` | 语法高亮 |
| 后端代码 | `.py`, `.go`, `.rs`, `.java`, `.kt`, `.swift`, `.rb`, `.php`, `.cs`, `.cpp`, `.c` | 语法高亮 |
| 脚本 | `.sh`, `.bash`, `.ps1`, `.bat`, `.cmd` | 语法高亮 |
| 配置/数据 | `.json`, `.jsonc`, `.yaml`, `.yml`, `.toml`, `.ini`, `.xml` | 语法高亮 |
| 图片 | `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.bmp`, `.svg`, `.ico` | 图片渲染 |
| 不支持 | `.pdf`, `.doc`, `.zip`, `.exe`, `.dll` 等 | 显示占位提示 |

#### 2.3.2 文本类文件查看

- **自动换行**：支持 `white-space: pre-wrap`，`word-break: break-all`
- **行号显示**：代码模式下左侧显示行号，不可选中，与代码垂直对齐
- **内容滚动**：支持垂直和水平方向滚动
- **文件大小限制**：2 MB，超出显示"文件过大"提示

#### 2.3.3 图片类文件查看

- **图片渲染**：使用 `<img>` 标签内嵌预览
- **SVG 处理**：作为图片渲染
- **无后缀文件**：尝试当文本读取

---

### 2.4 文件内容过滤功能

当前已实现基于文件类型的过滤机制：

- **前置过滤**：在发起文件读取请求前，先判断文件后缀是否在支持列表中
- **占位提示**：不支持的文件类型显示"暂不支持预览"提示
- **图标提示**：居中显示图标 + 文字说明

---

### 2.5 代码高亮能力

#### 2.5.1 技术方案

采用 **Shiki** 作为语法高亮引擎：

- **主题支持**：默认使用 `light-plus`（VS Code 默认浅色主题）
- **语言覆盖**：TypeScript、JavaScript、Python、Go、Rust、Java、CSS、HTML、JSON、YAML、Markdown、Bash、SQL 等
- **实例复用**：模块级别缓存，同一应用生命周期内只初始化一次

#### 2.5.2 集成要点

- **异步初始化**：应用启动时预初始化，避免阻塞 UI
- **降级处理**：高亮失败时静默回退到纯文本显示
- **行号集成**：行号与高亮代码并排显示，保持视觉一致性

### 2.6 查找与替换需求

#### 2.6.1 查找功能

- **快捷键**：`Ctrl+F` 打开查找面板
- **实时搜索**：输入时即时高亮匹配项
- **搜索选项**：
  - `Aa` 大小写匹配（`Alt+C`）
  - `W` 整词匹配（`Alt+W`）
  - `.*` 正则表达式（`Alt+R`）
- **结果导航**：上一个/下一个，环形循环，自动滚动居中
- **匹配计数**：显示当前项/总数（如 `3 / 12`）

#### 2.6.2 替换功能

- **快捷键**：`Ctrl+H` 打开替换面板
- **替换操作**：
  - 替换单个：替换当前聚焦项，自动跳转下一个
  - 全部替换：批量替换，需确认提示
- **正则支持**：支持捕获组引用（`$1`、`$2`）

#### 2.6.3 面板交互

- **关闭方式**：`Escape` 键或点击关闭按钮
- **高亮样式**：黄色背景高亮所有命中项，橙色背景标记当前项
- **错误处理**：正则表达式非法时显示红色边框和错误提示

### 2.7 Markdown文件支持

#### 2.7.1 渲染切换

支持三种视图模式：

| 模式 | 说明 | 触发方式 |
|------|------|----------|
| 代码模式 | 显示原始文本，带行号和语法高亮 | 默认进入，或点击「代码」按钮 |
| 渲染模式 | Markdown 渲染预览 | 点击「渲染」按钮 |

**切换按钮特点：**
- 仅 Markdown 文件显示
- 显示目标模式名称（当前代码→显示"渲染"，当前渲染→显示"代码"）
- 模式状态不持久化，每次打开新文件默认进入代码模式

#### 2.7.2 Markdown 文件导出

支持导出为两种格式：

| 格式 | 实现方式 | 特点 |
|------|----------|------|
| PDF | Electron `webContents.printToPDF()` | 渲染结果与预览完全一致，含代码高亮 |
| Word (.docx) | `docx` 库生成 | 结构化文档，支持标题、列表、代码块等 |

**导出流程：**
1. 点击「导出为...」按钮弹出下拉菜单
2. 选择目标格式
3. 弹出系统"另存为"对话框（默认路径为源文件目录）
4. 执行导出并显示成功/失败提示

## 3. 与详情面板（右栏）的关系

### 3.1 架构层次

```
DetailPanel（详情面板容器）
└── FileOverlay（文件浮层，selectedFile 非空时显示）
    ├── FileToolbar（工具栏）
    └── FileContentView（文件内容查看区）
        ├── 代码模式（行号 + 语法高亮）
        └── 渲染模式（Markdown 渲染）
```

### 3.2 交互联动

- **文件打开**：点击文件卡片时，`selectedFile` 赋值，浮层显示
- **文件关闭**：点击关闭按钮时，`selectedFile` 置为 `null`，浮层隐藏
- **状态管理**：`selectedFile` 通过 `AppContext` 在组件间共享

### 3.3 详情面板信息

当前详情面板主要显示文件内容，文件元数据（大小、创建时间、路径等）可通过工具栏操作间接获取。
