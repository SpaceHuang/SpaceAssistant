## 1. 产品概述

### 1.1 产品定位

SpaceAssistant 是一款基于 Electron + React + TypeScript 的跨平台桌面应用，集成 Anthropic Claude API 提供智能 Agent 能力。产品旨在为用户提供高效、安全、可定制的 AI 辅助交互体验。

### 1.2 目标用户

- 开发者与技术人员：需要 AI 辅助代码编写、技术文档生成
- 内容创作者：需要 AI 辅助写作、创意生成
- 专业人士：需要 AI 辅助数据分析、报告撰写
- 普通用户：寻求智能对话助手服务

### 1.3 平台支持

| 平台 | 最低版本要求 | 支持状态 |
|------|-------------|---------|
| Windows | Windows 10 64位 | 支持 |
| macOS | macOS 10.15+ | 支持 |
| Linux | Ubuntu 18.04+/Fedora 34+ | 支持 |

## 2. 技术架构

### 2.1 技术栈选择

| 层级 | 技术 | 版本要求 | 选型理由 |
|------|------|---------|---------|
| 桌面框架 | Electron | 25+ | 跨平台能力成熟，社区活跃，支持原生功能调用 |
| 前端框架 | React | 18+ | 生态成熟，性能优秀，支持 Hooks 和并发特性 |
| 类型系统 | TypeScript | 5+ | 提供类型安全，减少运行时错误，提升开发效率 |
| 状态管理 | Redux Toolkit | 2+ | 状态管理清晰，支持中间件扩展，开发体验好 |
| UI 组件库 | Ant Design | 5+ | 组件丰富，设计规范统一，支持主题定制 |
| HTTP 客户端 | Axios | 1+ | 功能完善，支持拦截器、请求取消，错误处理友好 |
| 图标库 | Lucide React | 最新 | 图标精美，支持自定义，体积小 |

### 2.2 架构设计

采用经典的 Electron 三进程架构：

```
┌─────────────────────────────────────────────────────────────────┐
│                      Main Process (Node.js)                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐    │
│  │ Claude SDK  │  │  Keytar     │  │   File System      │    │
│  │ Integration │  │ (密钥存储)   │  │   Operations       │    │
│  └─────────────┘  └─────────────┘  └─────────────────────┘    │
└──────────────────┬──────────────────────────────────────────────┘
                   │ IPC (contextBridge)
                   ▼
┌─────────────────────────────────────────────────────────────────┐
│                     Preload Script                              │
│          安全的 API 桥接层，类型定义与验证                        │
└──────────────────┬──────────────────────────────────────────────┘
                   │ window.api.*
                   ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Renderer Process (Browser)                   │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐    │
│  │   React     │  │   Redux     │  │   UI Components    │    │
│  │   Components│  │   Store     │  │   (Ant Design)     │    │
│  └─────────────┘  └─────────────┘  └─────────────────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 核心模块划分

| 模块 | 职责 | 所属进程 |
|------|------|---------|
| `chat-service` | 聊天消息管理、流式响应处理 | Renderer |
| `config-service` | 配置管理、API Key 安全存储 | Main + Renderer |
| `file-browser` | 本地文件系统浏览、预览 | Main + Renderer |
| `search-service` | 全局搜索、历史记录管理 | Renderer |
| `claude-client` | Claude API 封装、请求处理 | Main |
| `preload-api` | IPC 桥接、类型安全接口 | Preload |


## 5. 功能复用说明

### 5.1 可复用模块识别

从现有 NovelAI 项目中可抽取以下功能模块：

| 模块名称 | 源文件路径 | 复用程度 |
|----------|-----------|---------|
| Claude API 封装 | `electron/anthropicClientFactory.ts` | 高 |
| API Key 安全存储 | `electron/claudeKeytar.ts` | 高 |
| 流式响应处理 | `electron/claudeToolLoopStreamParams.ts` | 中 |
| IPC 桥接层 | `electron/preload.ts` | 中 |
| 类型定义 | `src/shared/*` | 高 |

### 5.2 复用策略

1. **直接复用**：类型定义、工具函数等无依赖模块可直接复制使用
2. **适配复用**：需要根据新项目架构进行接口适配的模块
3. **重构复用**：需要进行代码重构以符合新项目规范的模块

### 5.3 重构要点

- 将原项目中的叙事引擎相关代码剥离
- 移除 StorySwarm 相关功能
- 简化状态管理逻辑
- 适配新的 UI 组件库（Ant Design）

## 7. 安全与隐私

### 7.1 数据安全

| 安全措施 | 描述 |

| 措施 | 描述 |

| 本地存储 | 所有数据存储在本地，不上传云端 |
